# Getting started

Shandapha is a UI platform, not just a component pack. The fastest path is:
preview a pack, tune the brand kit, pick templates, and export either a starter
or a reversible patch for an existing project.

## Recommended flow

1. Open Studio and choose whether you are starting fresh or patching an
   existing app.
2. Select the target runtime: Next.js App Router, React with Vite, Web
   Components, or Blazor via Web Components.
3. Apply a style pack, then tune semantic brand inputs like primary, accent,
   font, radius, and density.
4. Choose templates before heavy modules. Templates are the real acceleration
   surface.
5. Run the generator doctor checks after export to confirm theme assets,
   providers, and patch safety.

## What stays portable

- `tokens.json` stays user-owned.
- CSS variables remain the runtime source of truth.
- Web Components remain the universal baseline.
- React and Next.js get the best DX through adapters, not rewrites.
